1.0.0: 柱体巡线避障版_形态过滤、分区回归、虚拟地图推演
本版本提供了一套基于 Maix 的自动避障小车视觉方案，专为解决K题赛道中“白柱、灰底、黄/白边墙”极易融为一体，且传统基于坐标的圆形检测极其耗费算力、极不稳定的痛点而设计。

系统彻底摒弃了传统的寻圆逻辑，采用了“垂直形态学过滤”与“局部线性回归”相结合的轻量化架构，将复杂的三维柱体避障巧妙地降维打击为高效的“巡线”逻辑，在保证极高抗干扰能力的同时，最大化释放了边缘设备的算力。

1. 智能形态过滤与分区回归机制
系统将画面从中心一分为二（左半区/右半区），独立进行评估与追踪：

形态学防火墙（排雷区）： 引入自定义 is_pillar() 校验逻辑，严格筛查色块长宽比（高度必须大于宽度的1.5倍）及最大宽度上限。自动拦截赛道横线、远端边墙以及倒地的柱体，精准剥离出“直立柱体”。针对赛道远景，自动屏蔽屏幕上方1/3区域的干扰。

降维回归车道（红框跟踪）： 当目标通过形态校验后，系统不再计算圆心坐标，而是框选该目标的ROI区域，直接调用 get_regression（线性回归）进行图像处理。输出目标相对于镜头的角度（theta）和偏移量，将“找柱子”转化为小车最擅长的“巡线”控制，实现极速处理。

惯性防抖锁定： 系统跨帧记录上一帧左右两区最优目标的中心坐标。若当前帧目标出现在上一帧坐标附近（动态关联），将直接赋予其 1.2 倍的权重加分，彻底解决多柱体重叠时焦点反复横跳的“闪烁”问题。

2. 虚拟地图自动修正与锁定机制（核心更新）
系统内部构建了一个 3x3 的赛道相交点坐标系与车位状态机（包含行、列、朝向），并拥有一套严密的全自动地图动态修正与锁定逻辑：

三重复检触发条件： 视觉系统不会盲目修改地图。只有同时满足以下三个条件，修正程序才会启动：

时间锁（防抖）： 小车执行前进或转向动作后，会自动进入 1 秒钟的“修正死区”，期间禁止一切地图修改，彻底避开车体晃动和变道时的视觉残影。

方向锁（校准）： 小车车体朝向必须为正北（heading == 0）。

边界锁（防越界）： 计算出的目标坐标必须在 3x3 的合法网格交点范围内。

精准空间坐标映射： 满足触发条件后，系统根据当前车位 (row, col) 自动换算前方的柱子坐标：

左半区识别到柱子，直接映射为左前方交点：(row - 1, col - 1)

右半区识别到柱子，直接映射为右前方交点：(row - 1, col)

黑柱属性永久锁定： * 当对应位置被识别为 T1（黑柱） 时，系统将该交点强转为黑色（值设为 2），并将其坐标写入永久锁定名单（_pillar_locked）。一旦锁定，该节点在后续比赛中将具备最高权重，绝不会被任何白柱或误判信号覆盖。

当对应位置识别为 T2（白柱）或前方无障碍 时，若该位置未被黑柱锁定，系统会将其动态修正或维持为白色（值设为 1），供后续路径规划使用。

3. 动态色彩锚定与环境持久化
触屏抗干扰取色： 依托 320x240 分辨率的可视化 GUI，用户可直接点击屏幕目标获取 RGB 像素并转化为 LAB 色彩空间。针对赛道灰底白柱的痛点，底层的 mk_threshold 函数对高亮度颜色（L >= 60）进行了极其严苛的容差收紧（明度±12，色彩±6），确保白柱与灰底/白墙彻底剥离。

配置断电保存： 获取的颜色阈值（T1/T2）会自动写入设备的 app_config_kv 配置文件中。关机重启后系统会自动读取上一场比赛的参数，实现断电持久化。

4. 双状态可视化调试界面
双状态交互界面： 提供直观的画面切换，通过底部“取阈值/二值化”以及顶部的“[地图/+]”按钮，无缝切换原图巡线模式与虚拟沙盘模式。彩色状态反馈明确：锁定目标绘制红色包裹框与回归线，双准星持续标记取色锚点（C1红/C2绿）。

⚠️ 开发者实车部署备注 (Release Notes)
单色策略建议： 虽然视觉系统具备 T1/T2 双色独立扫描能力，但在实际闭环控制中，强烈建议仅使用 T1 锁定一种关键色（如黑柱），将另一种颜色（白柱）视为默认背景障碍。这能大幅降低环境光干扰，提升系统帧率。

二值化显示 Bug 预警： 当开启双颜色阈值同时检索时，点击“二值化”预览可能会触发全白/过曝的渲染 Bug。此 Bug 仅属于图传显示层面的叠加冲突，不影响底层识别算法的正常运行和数据输出。

地图推演的通讯接管： 目前地图模式下的“转向/前进”由屏幕 GUI 按钮模拟推演。实际装车联调时，必须通过串口通讯截获主控板发出的物理运动指令（如：小车向左转弯、直行一个单位），来替代触摸事件，完成虚拟地图与真实车体坐标的同步。

死区与中断时间匹配： 代码中设定的 _correct_locked_until（地图修正后 1000ms 内禁止重复判断）是为了防止小车在转向过程中发生视觉误判。该数值为预设基准，实际比赛时必须根据小车底盘的旋转角速度、刹车性能进行联调缩放，找到最契合的视觉盲区时间。
